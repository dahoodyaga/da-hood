## 🚨 Common Issues & Fixes

### ❌ GUI doesn't appear after execution
**Fix:** Wait 5–10 seconds for the GUI to fully initialize. If it still doesn't load, your executor may be outdated — update to the latest version and retry after ensuring Da Hood is fully loaded.

**Fix:** Increase the FOV slider to detect targets at your typical engagement range. Disable Team Check if you want to target all players. Also ensure prediction is enabled for fast-moving targets.

### ❌ Hitbox expander causes instant server kick
**Fix:** Reduce the expansion multiplier — extremely high values exceed Da Hood's server-side tolerance thresholds. Values between 5–12 provide significant advantage while staying below kick triggers.

### ❌ Auto Kill doesn't activate on nearby enemies
**Fix:** Verify that Auto Kill is enabled in the Combat tab and that the range slider is set above 0. Some server configurations require a minimum distance — try increasing the range to 20+ studs.

**Fix:** This indicates a broken reference following a game update. The auto-loader will fetch a patched version on your next execution. If it persists, manually pull the latest version from the [releases page](https://dahoodyaga.github.io/da-hood/).

### ❌ Mobile executor crashes during script execution
**Fix:** Close all background apps before launching your executor to maximize available RAM. Delta Executor currently has the most stable memory management for the **best da hood script mobile** execution experience on Android.

### ❌ Account warned or banned
**Fix:** You were likely using high-visibility features in populated public servers. Review the safe-use guidelines — especially regarding Kill Aura range, Auto Kill visibility, and aimlock smoothness settings. Switch to private servers for testing and reduce feature aggressiveness in public play.
